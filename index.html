Descubre el mundo de las APIs

Las APIs son el puente invisible que conecta aplicaciones, servicios y datos en toda la web. Aprende cómo funcionan y por qué son esenciales en el desarrollo web moderno.

¿Qué es una API?

API significa Application Programming Interface (Interfaz de Programación de Aplicaciones). Es un conjunto de reglas y protocolos que permite que dos aplicaciones se comuniquen entre sí.

Imagina que estás en un restaurante: tú eres el cliente, la cocina es el servidor, y el mesero es la API. Tú le dices al mesero lo que quieres (petición), él va a la cocina, y te trae tu comida (respuesta). ¡Así de simple funciona una API!

🔗

Conectividad

Permite que diferentes aplicaciones y servicios se comuniquen sin importar el lenguaje en que fueron creadas.

⚡

Eficiencia

Evita reinventar la rueda. ¿Necesitas mapas? Usa la API de Google Maps. ¿Clima? Usa OpenWeatherMap.

🔒

Seguridad

Las APIs exponen solo los datos necesarios, manteniendo protegida la lógica interna del servidor.

¿Cómo funciona una API?

El proceso se basa en un modelo de petición y respuesta (Request/Response). El cliente envía una solicitud HTTP al servidor a través de la API, y el servidor devuelve los datos solicitados, generalmente en formato JSON.

Figura 1 — Flujo de comunicación: Cliente → API → Servidor → API → Cliente

Cada vez que usas una app de clima, inicias sesión con Google, o ves un mapa embebido en una página web, estás usando una API sin darte cuenta.

Métodos HTTP en las APIs

Las APIs REST utilizan los métodos HTTP para indicar qué acción se desea realizar sobre un recurso. Los cuatro principales se conocen como CRUD (Crear, Leer, Actualizar, Eliminar).

Figura 2 — Los métodos HTTP: GET (leer), POST (crear), PUT (actualizar), DELETE (eliminar)
Método Acción Ejemplo
GET Obtener datos del servidor Ver la lista de estudiantes
POST Enviar datos nuevos al servidor Registrar un nuevo estudiante
PUT Actualizar un recurso existente Cambiar el nombre de un estudiante
DELETE Eliminar un recurso Eliminar un estudiante de la lista

Tipos de APIs

Existen diferentes arquitecturas y protocolos para construir APIs. Las más populares en la actualidad son REST, GraphQL y SOAP. Cada una tiene sus ventajas dependiendo del proyecto.

APIs Públicas

Disponibles para cualquier desarrollador. Ejemplo: la API de Twitter, PokéAPI, o NASA.

🏢

APIs Privadas

Solo para uso interno de una organización. Conectan los sistemas internos de la empresa.

🤝

APIs de Socios

Compartidas entre empresas aliadas. Ejemplo: pasarelas de pago como PayPal o Stripe.

Formato JSON

JSON (JavaScript Object Notation) es el formato estándar que usan las APIs para enviar y recibir datos. Es ligero, fácil de leer tanto para humanos como para máquinas.

Figura 4 — Ejemplo de una respuesta JSON típica devuelta por una API

Un objeto JSON se compone de pares clave : valor encerrados entre llaves { }. Los valores pueden ser textos, números, booleanos, arreglos u otros objetos.

Ejemplo práctico con JavaScript

Aquí tienes un ejemplo real de cómo consumir una API usando fetch() en JavaScript. Este código obtiene datos de la API pública JSONPlaceholder:

// Consumir una API con fetch()
fetch('https://jsonplaceholder.typicode.com/users/1')
  .then(response => response.json())
  .then(data => {
    console.log('Nombre:', data.name);
    console.log('Email:', data.email);
    console.log('Ciudad:', data.address.city);
  })
  .catch(error => {
    console.error('Error:', error);
  });

Este código envía una petición GET a la API, recibe los datos en formato JSON y los muestra en la consola del navegador. Puedes probarlo en CodePen o en la consola de tu navegador.
